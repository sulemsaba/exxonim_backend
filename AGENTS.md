## Project Overview

## Architecture

FastAPI async REST API. Typical layout:
- `app/routers/` or `app/api/` — route handlers grouped by resource
- `app/models/` — Pydantic models (request/response schemas)
- `app/schemas/` — data validation schemas
- `app/services/` — business logic
- `app/db/` or `app/database.py` — database session and models

## Directory Structure

## Conventions

- Use Pydantic models for all request/response schemas — never raw dicts
- Async endpoints (`async def`) for I/O-bound operations
- Dependency injection via `Depends()` for auth, DB sessions, config
- HTTP exceptions: raise `HTTPException` with correct status codes
- Environment variables: use `pydantic-settings` or `python-dotenv` — never hardcode

## Agent Workflow

1. Check `app/routers/` for existing patterns before adding new endpoints
2. Add Pydantic schemas for every new endpoint request/response
3. Keep routers thin — delegate to services
4. Auto-generated docs at `/docs` (Swagger) and `/redoc` — verify new endpoints appear there
5. Use type hints everywhere — FastAPI relies on them for validation
